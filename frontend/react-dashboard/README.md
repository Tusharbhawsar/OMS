# Outage POC React Dashboard

React + TypeScript dashboard for the Phase 1 FastAPI outage communication POC.

## What this dashboard supports

- Upload Excel/CSV outage data
- View active outage KPIs
- View active planned outage rows
- Identify affected customers by outage
- Show Medical Baseline customer priority
- Process single planned outage notification workflow
- Run planned outage batch manually
- View SMS/Email sandbox notification logs (message text is AI-generated by Gemini and may be
  multilingual, e.g. English/Spanish, based on each customer's `preferred_language`)

## Backend expected

Run the FastAPI backend on port `8000` with this base API:

```bash
http://localhost:8000/api/v1
```

The backend CORS is already open in the provided POC code, so the React app can call it from Vite on port `5173`.

## Local setup

```bash
cd outage-poc-react-dashboard
cp .env.example .env
npm install
npm run dev
```

Open:

```bash
http://localhost:5173
```

## Environment

```bash
VITE_API_BASE_URL=http://localhost:8000/api/v1
```

## Recommended backend startup order

From the backend project:

```bash
cp .env.example .env
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload --port 8000
```

Then run this React app.

## API endpoints used

- `POST /uploads/outage-data`
- `GET /dashboard/summary`
- `GET /dashboard/active-outages`
- `GET /outages/active`
- `GET /outages/{outage_id}/affected-customers`
- `POST /outages/{outage_id}/process-planned?notification_type=Advance%20Notice`
- `POST /outages/batch/process-planned`
- `GET /notifications?limit=200`
- `GET /notifications?outage_id={outage_id}&limit=200`
